SAE 2.04

BUT1 Integrative Project: complete synthesis of skills in Networking, Telephony, IoT, and Web Development.

SAE Summary

Objective: This integrative project aims to bring together the knowledge and projects of the first year within a single infrastructure. It involves setting up a complete IT solution for a company, covering the local network, IP telephony, data collection via sensors, and their display on a web interface.

The project is divided into four major divisions: Networking, Telephony, IoT Collection, and Web/Database.

View source code on GitHub

Environment and Tools

Cisco Infrastructure

Configuration of physical routers and switches for the core network (VLANs, Routing, NAT).

Python

MQTT acquisition scripts and business logic for sensor data processing.

Django

Web framework used to create the dynamic dashboard and the data retrieval API.

MariaDB

Relational database system centralizing temperature readings.

Debian

Operating system for the virtualized servers hosting the scripts and the website.

Linphone

Softphone software used to test SIP connectivity across the different VLANs.

Infrastructure Architecture

Main technical steps

1. Network Segmentation

Implementation of 4 VLANs (Voice, Users, Server, Administrator) and configuration of inter-VLAN routing.

2. System Infrastructure Security

Implementation of ACL filtering on the router and strict iptables rules on the Debian server.

3. VoIP Communication

Registration of softphones and physical SIP phones, and configuration of network connectivity for voice streams.

4. MQTT Collection (Python)

Development of the MQTT broker listening script and automation of MariaDB database insertion.

5. Web Development (Django)

Design of the dynamic website managing the real-time display of collected data.

6. Service Deployment

Apache2 production deployment, NTP synchronization, and centralized logging via Rsyslog.

My role in the project

During this group SAE, my role was to retrieve temperature data via MQTT and display it on a website. For data collection, I used a Python script on a Debian VM, which ran continuously and was connected to a MariaDB database. This database was itself linked to a dynamic website created using Django whose goal was to display all the data.

Database Organization

The MariaDB database structures information to identify sensors and record readings over time.

Dynamic Dashboard (Django)

  • AJAX Refresh: Asynchronous requests every 5 seconds to update data without reloading the page.
  • Graphical Visualization: Integration of Chart.js to track the temporal evolution of the sensors.
  • Business Filtering: Dynamic sorting by sensor ID, name, or date range via Django views.
  • Data Export: Feature allowing the download of filtered records in CSV format.
