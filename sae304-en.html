SAE 3.04

Discovering pentesting: introduction to cybersecurity via Capture The Flag (CTF) challenges.

SAE Summary

The goal of this SAE was to introduce us to pentesting through CTF (Capture The Flag) challenges. The objective of these challenges is to obtain a text string (the flag) by performing a task similar to, for example, exploiting a flaw in a deliberately vulnerable environment.

We had access to the pro version of Root-Me, which offers a path adapted to the Networks and Telecommunications Bachelor's degree (EDU Path | Core curriculum). This path consists of 18 modules, grouping a total of 75 challenges allowing us to discover various introductory topics on aspects of cybersecurity.

Environment and Tools

Root-Me

Learning platform dedicated to hacking and information security offering vulnerable environments.

Kali Linux

GNU/Linux distribution oriented towards security and pentesting, used to execute various analysis and exploitation tools.

Python

Scripting language used to automate complex tasks or interact with TCP sockets.

Wireshark

Network packet analyzer used to dissect PCAP files and retrieve sensitive information.

Web Tools

Understanding HTTP requests, manipulating cookies, source code, and exploiting flaws like SQLi or XSS.

Linux System (Bash)

Use of basic commands, permission management, and privilege escalation in a Bash environment.

Progress on the platform

Topics Covered

1. Web Vulnerabilities

Introduction to web application security. Exploitation of classic vulnerabilities such as SQL injections (SQLi), XSS flaws, or authentication bypass.

2. Programming and Scripting

Creation of automation scripts. Use of TCP sockets in Python to dynamically interact with remote services to solve mathematical or logical problems within a time limit.

3. Network Analysis

In-depth study of network captures (PCAP files) using Wireshark. Understanding encapsulation, protocol analysis, and extracting cleartext data such as credentials.

4. Linux System

Command-line navigation, understanding the Linux permission structure, and exploiting misconfigurations to perform privilege escalations.

Validated Skills

This SAE is fully in line with the Cybersecurity track of the Bachelor's degree and validates the following learnings:

  • AC21.04 Exploit the vulnerabilities of a computer system or a web application.
  • AC22.01 Identify the threats and vulnerabilities of a system (flow and code analysis).
  • AC13.02 Read, execute, correct, and modify a program (automation scripting).
  • AC13.01 Use a computer system and its command-line tools (Kali Linux, Bash).

Reflective Analysis

During this SAE, I was able to consolidate my foundations in many areas and discover new concepts because these challenges rely on specific techniques. This approach invites research into documentation as well as mutual aid while studying, and I appreciated the very concrete approach of this SAE. During it, I was able to complete all of the 75 proposed challenges.
