SAE 2.03

Creation of a dynamic Web application: design and development of a movie database with CRUD architecture and the Django framework.

SAE Summary

Objective: The goal of this project was to set up a dynamic website (in my case, a movie database) relying on a SQL database. For the website development, I used the CRUD (Create Read Update Delete) principle with the Django framework for Python.

Acquired skills: Website development, use of Django, modeling and use of SQL queries, and setting up virtual machines for deployment.

View source code on GitHub

Environment and Tools

Python

Main programming language used for business logic and the project's backend.

Django

High-level Python Web framework allowing rapid development and respecting the MVT (Model-View-Template) architecture.

HTML5

Markup language used to structure the views (templates) and user interface of the application.

VirtualBox

Virtualization tool used to create virtual machines (Linux VMs) to deploy and host the web server.

MariaDB

Relational Database Management System (RDBMS) used for storing application tables in the final project.

Database Modeling

Before starting the implementation in Django, the first step was to model the data structure via a relational schema, corresponding to the needs of a movie database (referencing movies, categories, actors, and user comments management). The application ultimately relies on a MariaDB database.

User Interface and Forms

The site interface was designed to allow users to interact with the movie database. The site integrates multiple forms offering the ability to add categories, movies, users, actors, and of course comments.

Template Organization (MVT Architecture)

To respect Django's Model-View-Template architecture, I organized the project around reusable HTML templates. The templates folder contains 5 subfolders corresponding to the entities, each implementing CRUD operations.

index.html

Root file serving as the main skeleton for the application.

  • Project home page.
  • Contains the navigation menu common to all pages.
  • Contains an access link to the movie list page.
  • Serves as a parent template (via Django block tags) for the other views.

affiche.html

Displays detailed information of a specific entity (e.g., movie details). (Not necessary for all entities).

ajout.html

Contains the form allowing the Creation (Create) of a new entity. Necessary for all CRUDs.

modifier.html

Allows Updating (Update) the information of an existing entity. Based on the add form structure.

supprimer.html

Confirmation view allowing the Deletion (Delete) of an entity. Necessary for all CRUDs.

afficher_all.html

Allows Reading (Read) all entities of a table.

Can use an identifier (id) to filter the display (e.g., all movies of a genre, all comments of a movie...).

Project Management & Deployment

Planning with Gantt

The realization of the project began with a phase of structuring and task distribution, materialized by the creation of a Gantt chart. We had several study sessions and had to divide the work, which is why this planning method proved particularly useful for tracking progress.

User Management

A basic management method (without advanced security) was implemented: whenever a user wants to add, modify or delete a comment, they have the choice between creating their account or logging in, then accessing the comment form.

Deployment in Virtual Machines

Beyond local development, the project required setting up virtual machines (Linux VMs). The deployed architecture separates the services: an Apache server hosting the Web server (linked to Nginx and Gunicorn for Django) and a MariaDB server to host the database.

Targeted Critical Learnings

  • AC13.01 Use an IT system and its tools (deployment of virtual machines).
  • AC13.02 Read, execute, debug, and modify a Python/Django program.
  • AC13.04 Understand the architecture and technologies of a website (HTML, CSS, HTTP requests, MVT architecture).
  • AC13.05 Choose appropriate data management mechanisms (Relational SQL database and Django ORM queries).
  • AC13.06 Integrate into an environment conducive to development (tracking via Gantt chart and versioning).

Reflective Analysis and Difficulties

I found this SAE particularly interesting because it makes us use techniques ranging from project management to production deployment.

However, we faced several difficulties that impacted the project:

  • Organizational delay: Unforeseen difficulties led to a shift from the initial planning established in the Gantt chart.
  • Media management: Image management with Django (requiring modification of settings.py and urls.py) proved complex and ultimately did not work for displaying actor photos.
  • Implementation and deployment issues: After deploying the application on virtual machines, some internal links no longer worked, requiring last-minute corrections.
