SAE 2.01

Create a network for a small business: setting up a functional and secure network architecture via virtualization.

SAE Summary

Objective: Set up a computer network for a small business, using GNS3 software, allowing device virtualization.

During this project, I was able to acquire concrete skills in network administration, in particular the management of VLANs, network security through the implementation of ACLs (access control lists), as well as server management (DHCP, DNS...). I also discovered the use of GNS3 software.

Environment and Tools

GNS3

Complex network emulation software. It made it possible to virtualize the entire architecture (routers, switches, and servers) to faithfully reproduce the SME environment.

Cisco

Professional network equipment (routers and switches) emulated under GNS3. The entire architecture relies on configuring Cisco IOS for routing and switching.

Debian

GNU/Linux distribution used to simulate the servers in the topology. Chosen for its stability, it allowed network services to run efficiently within GNS3 in a simpler way.

Network Architecture

Main technical steps

1. Segmentation and VLANs

Creation of dedicated VLANs to logically separate flows (Administration, Developer, Administrator). Configuration of "Trunk" links (802.1Q) and security by modifying the default native VLAN (VLAN 999).

2. Addressing and Routing

Implementation of IP addressing on physical and virtual interfaces (SVI). Configuration of the dynamic routing protocol RIPv2 to ensure connectivity between the various routers in the topology.

3. Redundancy with STP

Deployment of the Spanning Tree Protocol (STP) to avoid network loops. Strategic configuration of priorities (Root Primary / Secondary) to load balance between switches.

4. Server Deployment

Installation and configuration of Debian servers: setup of a DHCP server (isc-dhcp-server) with relay (ip helper-address), Web servers (Apache2), and synchronized file transfers via Rsync.

5. Security (Port-Security & ACL)

Activation of Port-Security ("mac-address sticky" and "restrict" mode) on switches to prevent unauthorized physical connections. Creation of ACLs (Access Control Lists) to filter HTTP, SSH, and ICMP traffic.

6. External Access (NAT/PAT)

Configuration of address translation (dynamic NAT with overload / PAT) on the edge router to allow equipment on the private network to access the Internet securely.

Validated Skills
