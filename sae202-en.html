SAE 2.02

Creation of a portable Web Radio: microcontroller programming for internet audio stream access and remote control.

SAE Summary

Objective: The goal of this project is to use and program two stacked Arduino boards to access internet radio. I used an Android application to remotely control this web radio.

During this project, I gained skills in C++ programming, using the Arduino ESP32 board, and familiarizing myself with the Arduino IDE.

View source code on GitHub

Environment and Tools

ESP32 & VS1053b

ESP32 microcontroller (Wi-Fi module) paired with a VS1053b audio Codec board for hardware decoding of MP3 and AAC streams.

Arduino IDE

Development environment used for C++ programming of the microcontroller and integration of various network and audio libraries.

IoT MQTT Panel

Android application used to create a remote control interface (sliders, buttons) communicating via the MQTT protocol.

System Overview

Main Technical Steps

1. Hardware Communication

The ESP32 interacts with the VS1053b audio codec via the SPI (Serial Peripheral Interface) bus, a synchronous protocol using the MOSI, MISO, SCK pins, and control signals like XDCS.

2. Hardware Audio Decoding

The VS1053b supports direct decoding of MP3/AAC streams. Sound control is achieved by writing to its SCI registers, such as SCI_VOL (volume) and SCI_BASS (bass and treble).

3. Dynamic Wi-Fi Management

Implementation of the WiFiManager library. It allows the ESP32 to create a temporary captive portal if no known network is found, avoiding hardcoding Wi-Fi credentials into the program.

4. Stream Reading (Streaming)

Use of the ESP32_VS1053_Stream library to retrieve audio streams from web radios directly from their URLs and send them seamlessly to the decoder.

5. Keyboard Commands (Serial)

Creation of a local control interface via the USB serial console (115200 baud). Listening to the keyboard allows the user to change stations (n/v), adjust volume (y/b), tweak bass/treble, or enable spatialization (s).

6. Telemetry and MQTT Control

Implementation of the MQTT (Message Queuing Telemetry Transport) protocol, based on publish-subscribe (PubSub). It receives commands from the Android app and sends back the current state of the radio.

7. Spatial Processing

Activation of the codec's EarSpeaker function via the SCI_MODE register to simulate loudspeaker listening through headphones, and use of a plugin to integrate a 5-band equalizer.
